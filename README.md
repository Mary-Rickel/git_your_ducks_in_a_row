
# Putting Ducks in a Row!
Code repository for PDP 2026 team 12 activity.

This repository contains scripts to put images of ducks in a row. This code is useful for anyone who is a duck enthusiast and would like to organise their ducks. 

CODEBASE MANAGEMENT RULES:
All commits must be reviewed by a manager and approved before being merged into a main/production branch. All commits must pass all unit tests before submission for merge. All new features must receive new sets of unit tests to verify functionality. Merge comments must be descriptive, relevant, and exhaustive with intent to apply new functionality and ensure that interpretability is optimised. All code should be commented before commit. Organise everything in folders where appropriate to section off code and images properly. When a contributor leaves, it is expected to train one another in an exhaustive onboarding process. The TOP DOG manager is responsible for keeping a well trained ALPHA merger at all times to make sure that we have a running knowledge “homebase."
