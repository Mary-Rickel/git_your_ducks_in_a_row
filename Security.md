# Security Policy

## Supported Versions

We actively maintain and patch security vulnerabilities for the following versions of this project:


| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| 1.5.x   | :white_check_mark: |
| < 1.5.x | :x:                |

## Reporting a Vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

If you discover a security vulnerability within this project, please report it responsibly by following these steps:

1. Send an email to **SirJohnGitJrSireJr5th@hotmail.com**.
2. Include a detailed description of the vulnerability.
3. Provide step-by-step instructions or a proof-of-concept (PoC) to reproduce the issue.

### Our Response Process

* **Acknowledgement**: We will acknowledge receipt of your report within 48 hours.
* **Evaluation**: We will investigate the issue and keep you updated on our progress.
* **Fix**: If verified, we will develop and release a patch. 
* **Disclosure**: We will coordinate a public disclosure date with you to ensure users have time to update.

Thank you for helping keep this project secure!
